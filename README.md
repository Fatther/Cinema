# Система управления кинотеатром

**Система управления кинотеатром** — это бэкенд-сервис для администрирования кинопроката и автоматизации работы с репертуаром.

Проект обеспечивает структурированное управление ресурсами кинотеатра, поддерживает гибкую настройку афиши по заданным критериям и гарантирует надежность за счет строгого соблюдения архитектурных паттернов.

---
## Ключевые возможности

- **Управление репертуаром**: Полный цикл администрирования данных о фильмах, доступных для проката.
- **Гибкая настройка**: Динамический подбор контента для кинозалов на основе хронометража и возрастных ограничений.
- **Многослойная архитектура**: Разделение ответственности между контроллером, сервисом и репозиторием (Controller → Service → Repository).
- **Безопасность данных**: Использование DTO и мапперов для изоляции внутренней логики системы от внешних API-ответов.
- **Гарантия качества**: Интеграция с Checkstyle и SonarQube для соблюдения индустриальных стандартов чистого кода.
- **Мгновенный отклик**: Работа с данными в оперативной памяти (In-memory) для обеспечения максимальной скорости обработки запросов.

---
## Доменная модель

Центральным элементом системы является сущность **Фильм (Movie)**, которая содержит параметры, необходимые для формирования сетки вещания:

* **ID**: Уникальный идентификатор в системе.
* **Заголовок**: Название фильма для афиши.
* **Длительность**: Хронометраж в минутах (используется для планирования сеансов).
* **Возрастной ценз**: Категория доступа (например, 0+, 12+, 16+, 18+).
* **Жанр**: Тематическая категория произведения.

Взаимодействие с внешними потребителями происходит через **MovieDto**, что позволяет гибко изменять структуру ответа без влияния на базу данных.

---
## Жизненный цикл запроса

1. **Запрос**: Клиент обращается к системе через REST-эндпоинты.
2. **Маршрутизация**: Контроллер обрабатывает входные параметры (например, ID фильма в `@PathVariable` или фильтр в `@RequestParam`).
3. **Обработка**: Сервисный слой выполняет бизнес-логику и фильтрацию ресурсов.
4. **Трансформация**: Компонент Mapper преобразует внутренние сущности в формат, оптимизированный для клиента.
5. **Ответ**: Система возвращает структурированный JSON-объект и статус-код HTTP.

## Структура проекта

```text
pavel.lab.cinema
├── controller   # Точки входа API для управления кинотеатром
├── service      # Бизнес-логика и правила управления репертуаром
├── repository   # Слой хранения данных
├── entities     # Основные сущности
├── dto          # Объекты передачи данных
└── mapper       # Компоненты трансформации данных

```

## Запуск и использование

- Склонируйте репозиторий.
- Соберите проект:
```bash
./gradlew build
```
- Запустите сервис:
```bash
./gradlew bootRun
```
- **API** доступно по адресу: `http://localhost:8080/movies`
